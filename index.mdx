AI-Implement connects your Linear workspace to your GitHub repositories. Label any issue `AI-Implement`, and within minutes Claude Code checks out the repo, implements the ticket, opens a pull request, and posts a gap analysis comparing what was built to what was asked. Your team reviews in the tools they already use — nothing changes about how you ship.

## How it works in 60 seconds

<Steps>
  <Step title="Label an issue">
    Add the `AI-Implement` label to any Linear issue. Write a clear, focused ticket — that's your prompt.
  </Step>
  <Step title="Claude implements it">
    The orchestrator picks up the issue, dispatches a GitHub Actions workflow in your target repo, and Claude Code reads your codebase and implements the change.
  </Step>
  <Step title="Review the PR">
    A pull request appears in your repo with a gap analysis comment showing what was implemented and what (if anything) still needs attention.
  </Step>
  <Step title="Iterate with /ai-implement">
    Comment `/ai-implement` on the PR to send Claude back in for a gap-fill pass on the same branch.
  </Step>
</Steps>

## What makes it different

AI-Implement is built for teams, not individual developers. The ticket is the prompt, and your existing Linear backlog is the context. Every run produces a PR, a gap analysis, and a Linear status update — so reviewers always know exactly what was attempted and why.
